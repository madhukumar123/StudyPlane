Below is a **Unified Dependency Graph for Data Structures and Algorithms (DSA)**.
This map connects **data structures → algorithms → algorithm paradigms → complexity theory**, showing how each concept builds on previous ones.

Think of it as a **layered knowledge graph**, not a simple list.

---

# 10. Algorithm Design Paradigms

These are **meta-techniques for designing algorithms**.

## Divide and Conquer

```
Divide
Solve recursively
Combine
```

Examples:

* Merge sort
* Quick sort
* Strassen matrix multiplication

---

## Greedy Algorithms

Make the locally optimal choice.

Examples:

```
Activity selection
Huffman coding
Kruskal MST
Prim MST
```

---

## Dynamic Programming

Used when:

```
Optimal Substructure
+
Overlapping Subproblems
```

Classic problems:

```
Knapsack
Longest Common Subsequence
Matrix Chain Multiplication
Edit Distance
```

---

## Backtracking

Used for exhaustive search.

Examples:

```
N Queens
Sudoku
Permutations
Subsets
```

---

## Branch and Bound

Optimization improvement over backtracking.

Used in:

```
Traveling Salesman
Integer programming
```
